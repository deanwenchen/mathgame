# 儿童算数小能手 - 游戏设计文档

## 一、世界观设定

### 1.1 故事背景

在遥远的**数学王国**，数字与符号和谐共存。王国的力量源泉是**智慧宝石**，它由历代数学大师守护。然而，**混乱魔王**觊觎宝石的力量，发动了入侵。魔王释放的"数字迷雾"让王国居民忘记了数学知识，整个王国陷入混乱。

**主角**：一位普通的学童，偶然获得了数学王国的求救信。为了拯救数学王国，主角化身为**算数勇士**，踏上冒险之旅。

### 1.2 核心主题

- **知识就是力量**：数学能力直接转化为战斗力量
- **成长与探索**：通过学习新知识解锁新地图和新能力
- **勇气与智慧**：面对困难，用智慧战胜敌人

### 1.3 世界地图

```
                    [几何迷宫] (5-6年级)
                         |
    [分数冰川] (4年级) ---+--- [方程塔] (6年级)
           |                          |
           +----------+---------------+
                      |
              [乘除法火山] (2-3年级)
                      |
              [加减法森林] (1年级)
                      |
                  [新手村]
```

## 二、角色设计

### 2.1 主角：算数勇士

| 属性 | 描述 |
|------|------|
| 职业 | 算数勇士 |
| 武器 | 数学法杖（初始）/ 计算宝剑（进阶） |
| 能力来源 | 数学知识 |

#### 成长属性
- **生命值 (HP)**：答错题目会扣血，HP归零则挑战失败
- **攻击力 (ATK)**：影响答对题目的伤害倍率
- **防御力 (DEF)**：减少答错时的伤害
- **暴击率 (CRIT)**：连击触发暴击的概率
- **智慧值 (WIS)**：经验值积累，用于升级

## 三、对战系统机制

### 3.1 核心战斗流程

```
┌─────────────────────────────────────────────────────────┐
│                      战斗界面                           │
├─────────────────────────────────────────────────────────┤
│  [怪物区域]                              怪物名称        │
│    /\___/\                              HP: ████░░░░   │
│   ( o o )                               Lv.5           │
│   (  v  )                                              │
│                                                         │
├─────────────────────────────────────────────────────────┤
│  [题目区域]                                            │
│   计算: 15 + 27 = ?                                     │
│                                                         │
│   [ 32 ]  [ 42 ]  [ 38 ]  [ 52 ]   <-- 选项按钮        │
│                                                         │
├─────────────────────────────────────────────────────────┤
│  [状态栏]                                              │
│   HP: ████████░░  80/100    ATK: 15    连击: x3        │
│   当前技能: 加法斩击                                     │
└─────────────────────────────────────────────────────────┘
```

### 3.2 答题机制

#### 正确答题
| 事件 | 效果 |
|------|------|
| 基础伤害 | 对怪物造成 `ATK × 题目难度系数` 点伤害 |
| 连击奖励 | 连续答对3题，触发**暴击**（伤害×1.5） |
| 连续5题 | 触发**超级连击**（伤害×2.0） |
| 快速答题 | 5秒内答对，额外伤害加成20% |

#### 错误答题
| 事件 | 效果 |
|------|------|
| 基础惩罚 | 扣除 `10 + 怪物等级×2` 点HP |
| 连击中断 | 连击计数归零 |
| 怪物攻击 | 怪物对玩家造成一次攻击 |

### 3.3 难度系数表

| 题目类型 | 难度系数 | 年级范围 |
|----------|----------|----------|
| 20以内加减法 | 1.0 | 1年级 |
| 100以内加减法 | 1.2 | 1-2年级 |
| 表内乘除法 | 1.5 | 2-3年级 |
| 两位数乘除法 | 1.8 | 3-4年级 |
| 分数运算 | 2.0 | 4-5年级 |
| 小数运算 | 2.2 | 4-5年级 |
| 简单方程 | 2.5 | 5-6年级 |
| 几何计算 | 2.8 | 5-6年级 |

### 3.4 技能系统

玩家通过正确答题积累**技能能量**，能量满后可释放强力技能。

| 技能名称 | 解锁条件 | 效果 | 能量消耗 |
|----------|----------|------|----------|
| 加法斩击 | 初始技能 | 基础伤害+10% | 0 |
| 减法护盾 | 击败10只减法怪 | 下次答错伤害减半 | 20 |
| 乘法风暴 | 掌握乘法口诀 | 下一题伤害×2 | 30 |
| 除法审判 | 通过乘除法火山 | 直接削减怪物20%HP | 40 |
| 分数裂变 | 通过分数冰川 | 连续3题必暴击 | 50 |
| 几何结界 | 通过几何迷宫 | 10秒内答错不扣血 | 60 |

### 3.5 战斗结算

#### 胜利奖励
- **经验值**：根据怪物等级和战斗表现计算
- **金币**：用于购买道具和装备
- **装备掉落**：概率获得装备碎片
- **知识点解锁**：首次通关解锁相关教学动画

#### 失败处理
- HP归零则战斗失败
- 可使用**复活道具**继续战斗
- 或返回城镇恢复后重新挑战

## 八、UI/UX设计要点

### 8.1 界面原则

### 9.2 核心数据结构

```typescript
// 怪物数据结构
interface Monster {
  id: string;
  name: string;
  hp: number;
  maxHp: number;
  attack: number;
  questionType: QuestionType;
  specialAbility?: Ability;
  sprites: {
    idle: string;
    attack: string;
    hurt: string;
    death: string;
  };
}

// 题目数据结构
interface Question {
  id: string;
  type: QuestionType;
  difficulty: number;
  content: string;
  options: string[];
  correctAnswer: string;
  timeLimit: number;
}

// 战斗状态
interface BattleState {
  player: PlayerState;
  monster: Monster;
  currentQuestion: Question;
  combo: number;
  skillEnergy: number;
  battleLog: BattleLogEntry[];
}
```

### 9.3 游戏循环

```
1. 初始化战斗场景
2. 加载怪物数据
3. 生成题目
4. 等待玩家答题
```
